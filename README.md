# Reegle ジラフユニット 商談プレゼンシステム

オンライン商談（Zoom/Teams等）で**台本を見ながら投影スライドを進める**ためのシステム。
3タイプ（自治体／幼稚園／写真スタジオ）切替対応。

---

## ファイル構成

| ファイル | 用途 |
|---|---|
| **present.html** | 進行用（自分用）。台本＋プレビュー＋切替操作 |
| **index.html** | 投影用（先方に見せる）。スライドだけのクリーン画面 |

両方ともダブルクリックで開ける（file://で動く）。サーバ不要。

---

## 商談中の使い方

1. **`present.html` をブラウザで開く**（自分のPC、Chrome/Edge推奨）
2. **上部のタイプ切替で対象を選ぶ**（自治体／幼稚園／写真スタジオ）
3. **右上の「🖥 投影画面を別タブで開く」をクリック**
   → 別タブに投影用画面が開く（＝先方に見せる用）
   → ボタン横の表示が「投影画面：接続中」（緑）になる
4. **Zoom/Teamsで「画面共有」→ その別タブ（投影用）だけを共有**
   - 「全画面共有」ではなく**ウィンドウ単位の共有**を選ぶ
   - 投影用タブで F11 を押すと全画面化されてより綺麗
5. **手元の present.html で：**
   - 左ペインの台本ブロックをクリック → 投影画面が連動
   - キーボード「← →」でスライド送り
   - 「次へ」「前へ」ボタンでも操作可
   - タイプ切替も即時に投影画面へ同期

**重要：** 別タブ（投影用）には台本は表示されません。先方からは絶対に台本が見えません。

## 設計上のポイント

### 共通スライドとタイプ別スライドのハイブリッド
13スライド構成のうち、**6スライド（サービス概要／3つのメリット／基本サービス／クロージング／次回日程／切り返し）は全タイプ共通**。残り6スライド（表紙／会社紹介／アイスブレイク／関心・ポジショニング／実績紹介／ヒアリング／Q&A）はタイプ別の内容。
進行用画面では共通スライドには「共通」タグが表示されます（投影用では非表示）。

### 投影用画面のフッター
タイプ名（自治体向け等）は**投影用には表示されない**（先方にセグメント露出しないため）。
「Reegle株式会社」のみ表示。進行用画面ではタイプ名も含めて表示されます。

### 同期の仕組み
ブラウザの BroadcastChannel API を使用。
- **必須条件**：同じPC・同じブラウザで2タブを開くこと
- 別端末・別ブラウザでは同期しません（同一オリジン制約）

## トラブルシューティング

- **「投影画面：未接続」のまま**：
  - ポップアップブロックを解除（右上のURLバーで許可）
  - もう一度「投影画面を別タブで開く」を押す
- **画像が表示されない**：
  - ファイルが破損している可能性。再ダウンロードしてください
- **古いブラウザで動かない**：
  - Chrome 88+、Edge 88+、Firefox 78+、Safari 14+ で動作確認推奨

## 編集したい場合

中身の編集には3ファイル必要（ソース込みのフルパッケージを別途用意可）：
- `slides_data.json` — スライドの内容（タイトル・本文・画像参照）
- `scripts.json` — 各タイプの台本テキスト
- `slides.css`、`renderer.js` — デザインとレンダラ

`bundle.py` で全部を `present.html`／`index.html` にインライン化する仕組み。
